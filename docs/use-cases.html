Use cases

Patterns and recipes built on Queen.

Queen on its own is a partitioned message broker. The same queen-mq npm package also ships a fluent stream-processing SDK — windows, joins, stateful reducers, and gates — that runs in your own Node process and commits state, sink emissions, and source acks atomically in PostgreSQL. The pages below are end-to-end recipes that combine the broker with the streaming SDK to solve real production problems.

Each "Live" card links to a dedicated page with code you can copy-paste and numbers from real stress runs. Cards marked "Coming soon" describe patterns the engine already supports today; their dedicated pages are being written next.

Canonical recipes

Live · backpressure

Rate limiter for external APIs

Throttle outbound calls to OTAs, LLM providers, or any third-party API with a token-bucket gate. Per-key state, FIFO order on retries, no deferred queue, scales linearly with multiple stream consumers.

.gate(tokenBucketGate({ capacity, refillPerSec, costFn }))
stress-tested: 9,431 msg/sec, 500k events, 0 errors

Real-time windowed aggregations

Tumbling, sliding, session, and cron-aligned windows. Maintain rolling sums, counts, p95s per key, and emit them to a sink queue or push them onto a live dashboard.

.windowTumbling({ seconds: 60 }).aggregate({ count, sum, max })

Cross-stream joins

Join two streams keyed by the same entity (reservations × messages, orders × payments, users × events) over a session window. Common pattern for proactive workflows and CDP-style guest profiles.

.keyBy(m => m.guestId).windowSession({ gap: '14d' }).reduce(merge)

Coming soon · safety

Anti-spam & anomaly detection

Per-sender velocity windows, urgency-keyword detection, sentiment-streak escalation. Routes flagged messages to a quarantine queue or fires escalation alerts within the second.

.keyBy(m => m.from).windowSliding({ size: '5m', hop: '30s' })

Coming soon · ETL

Continuous deduplication

Drop duplicate events arriving on flaky webhooks (Octorate, Booking, Stripe) by externalId within a short window. Per-partition state shard, exactly-once forward.

.windowTumbling({ seconds: 30 }).reduce(seenSet)

Coming soon · ops

Heartbeat & liveness monitoring

Each component publishes a heartbeat; an idle-flush stream detects missed beats per (component, host) and emits alerts. The trigger is the absence of events — only streaming engines handle this natively.

.windowTumbling({ seconds: 30, idleFlushMs: 5000 })
